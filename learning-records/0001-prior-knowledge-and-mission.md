# Starting Point: New to MCP, Python-focused, workshop goal

User is brand new to MCP with no prior exposure. They want to learn MCP end-to-end: understand the concept, build a Python server with FastMCP, then build an agent that uses it. Python is confirmed as the language of choice. This sets the floor at zero for MCP-specific knowledge — start from first principles in Lesson 1 before any code. The workshop framing (hands-on, goal-oriented) means every lesson should end with something runnable or testable.

**Implications**: Skip TypeScript examples. Keep JSON-RPC internals light in Lesson 1 — just enough to understand what's happening, not to implement it. The capstone (Notes MCP server + agent) should be introduced early so each lesson feels like progress toward a real thing.
